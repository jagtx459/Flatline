Each target defines a machine or service and exactly what runs on it when triggered. Credentials are encrypted at rest and never sent back to the browser — when editing, leave a credential blank to keep the stored value.

▶ On trigger: Flatline connects over SSH with these credentials and runs the command below on the host.

Set up a Linux account for this

Create a dedicated user that can run the shutdown command without a password prompt (so nothing privileged needs storing here). Run as root / with sudo on the target machine:

sudo useradd -m -s /bin/bash flatline
sudo passwd flatline               # set a password, or use an SSH key instead

# Let it run shutdown without a sudo password:
echo 'flatline ALL=(root) NOPASSWD: /sbin/shutdown, /usr/sbin/shutdown' \
  | sudo tee /etc/sudoers.d/flatline
sudo chmod 440 /etc/sudoers.d/flatline

# (recommended) add your Flatline SSH public key to
# /home/flatline/.ssh/authorized_keys  — chmod 700 the .ssh dir, 600 the file

Then set Username flatline and command sudo shutdown -h +0.

If the command needs sudo, the cleanest fix is passwordless sudo for it on the host (NOPASSWD: in sudoers) — nothing to store here. Otherwise set a sudo password below; Flatline writes it to the command's stdin on trigger, so the command must read it there itself (e.g. start it with sudo -S).

Optional restore command — run from the target's Restore button to undo the command above (e.g. cancel a pending shutdown). Runs over the same connection, with the sudo password above if set. Leave blank if there's nothing to undo.

▶ On trigger: Flatline signs in to the Windows host over WinRM with these credentials and runs the command below via remote PowerShell. Needs WinRM enabled on the host (Enable-PSRemoting -Force); the default port is 5985. Works with a local or domain account (NTLM).

Set up a Windows account for this

Recommended: create a dedicated local admin account rather than using a Microsoft account (whose identity over WinRM/NTLM is unreliable). Run in an elevated PowerShell on the target machine:

net user flatline * /add                     # create the user (prompts for a password)
net localgroup Administrators flatline /add  # WinRM requires admin rights
Enable-PSRemoting -Force                     # turn on WinRM / PowerShell remoting

# Workgroup (non-domain) machines only: let local accounts sign in remotely
# with full rights, otherwise you get "Access is denied":
New-ItemProperty -Path 'HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System' `
  -Name LocalAccountTokenFilterPolicy -Value 1 -PropertyType DWord -Force
Restart-Service WinRM

Then set Username flatline, leave Domain blank, and keep port 5985.

The command is run as a PowerShell command line. Optional restore command — run from the target's Restore button to undo the command above (e.g. cancel a pending shutdown). Leave blank if there's nothing to undo.

▶ On trigger: Flatline authenticates to the cluster API and performs the selected action below.

A raw Kubernetes API request: method + path (relative to the API server URL above) + optional JSON body. E.g. to scale a Deployment to 0: PATCH apis/apps/v1/namespaces/default/deployments/myapp/scale with body {"spec":{"replicas":0}}.

Optional restore request — run from the target's Restore button to undo the command above (e.g. scale back up). Leave blank if there's nothing to undo.

▶ On trigger: Flatline sends exactly this request (method, body, and auth below) — use it for webhooks or services with their own shutdown API.

Optional restore request — run from the target's Restore button to undo the request above. Uses the same auth as the trigger request. Leave blank if there's nothing to undo.
