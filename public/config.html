General

Encryption key

Stored credentials (passwords, SSH keys, tokens, webhook URLs) are encrypted with AES-256-GCM. Rotating re-encrypts everything under a fresh key in one step; nothing has to be re-entered. Back the key up, without it stored credentials are unrecoverable.

Backup & restore

Configuration (JSON)

Export endpoints, Flatline groups, action targets, action groups, and notification channels to a JSON file. Credentials are included but stay encrypted. Importing to another instance works only if it uses the same encryption key. Importing replaces all existing configuration.

Database

Download a full snapshot of the SQLite database (configuration and history), or restore one. Restoring replaces the entire database; stored credentials work only if this instance uses the same encryption key as the backup. It is STRONGLY recommended to have access to your current encryption key or generate a new one prior to performing a backup->restore procedure.

This will reset all endpoints, groups, actions, channels, history, removes the site password, and clears settings. Only the encryption key is kept.
